# Ubuntu 26.04 LTS WebServer (Nginx) 構築プロジェクト

仮想環境（VirtualBox）上にLinuxサーバーを構築し、Webサーバー（Nginx）の導入とホストOSからのアクセス検証を行いました。

## 構成環境
- **ホストOS**: Windows
- **ゲストOS**: Ubuntu Server 26.04 LTS (VirtualBox)
- **Webサーバー**: Nginx

## 実施内容
1. SSHによるリモート接続環境の開通（IP: 192.168.11.10）
2. Nginxのインストールおよびサービス起動確認
3. ホストOSブラウザからの疎通確認テスト

## アップグレード：Pythonによるサーバー自動監視システムの構築

### 1. 概要（要件定義）
サーバーのストレージ（ハードディスク）枯渇によるシステムダウンを防ぐため、サーバー自身が24時間自動でディスク残量を監視し、リアルタイムにWeb画面を更新し続ける「自動監視ツール」を構築しました。

### 2. システム構造（設計）
- **システム開発 (Python)**: `shutil` ライブラリを使用し、Ubuntuのディスク容量（全体・使用量・空き容量）を自動取得。結果を装飾したHTML（index.html）として動的生成するスクリプト（`disk_check.py`）を作成。
- **サーバー構築 (Nginx)**: Nginxのデフォルト設定を変更し、初期画面ではなくPythonが生成したカスタムHTML（監視パネル）をホストOS（Windows）へ公開するように再設計。
- **運用保守 (自動化/cron)**: Linuxの定時実行タスク（`cron`）を設定し、人間が介在することなく1分間隔でプログラムが自走し続ける完全自動化環境を構築。

### 3. テスト結果（エビデンス）
Windows側のブラウザからUbuntuのIPアドレスにアクセスし、サーバーのリアルタイムなストレージ容量が日本語の監視パネルとして正常に表示されることを確認しました。
