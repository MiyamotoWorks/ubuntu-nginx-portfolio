# VirtualBoxを使ったLinuxサーバ構築とWebサーバ運用・監視検証

## 概要
本プロジェクトは、VirtualBox上のLinux（Ubuntu Server）環境にWebサーバ（Nginx）を構築し、安全なリモート管理や運用確認を行ったものです。また、Pythonによる簡易ディスク監視ツールを組み合わせ、基盤SE業務における「サーバ構築・運用・セキュリティ対策・自動化」の基礎工程を一通り実証・検証しています。

---

## 1. 要件定義
- **Webサーバの安全な公開**: 仮想サーバ上にWebサーバを構築し、ホストOS（Windows）のブラウザから安全にアクセス可能とする。
- **セキュリティの確保**: 外部ネットワークからの不正アクセスを防ぐため、ネットワークの隔離と適切なポート管理を行う。
- **自動運用の共通化**: サーバのリソース状態（ディスク使用状況）を自動で定期取得し、可視化する簡易監視機能を実装する。

---

## 2. 設計

### 2.1 システム構成概要
本システムはセキュリティと実務的なインフラ設計を考慮し、**NAT構成（ポートフォワーディング）**を採用した以下の4層構造で設計されています。

1. **ホストOS（Windows 11 Pro）**
   - 開発・管理環境として使用。
   - ブラウザから `http://localhost:8080` にアクセスし、監視画面（WEB）を表示。
   - コマンドプロンプトからSSHクライアントを使用して、Ubuntuへの安全なリモート操作を実行。
2. **仮想化基盤（Oracle VirtualBox）**
   - **NATネットワーク構成（セキュリティ考慮）**
   - ゲストOSを外部ネットワークから隔離し、ホストOS経由の指定ポートのみ通信を許可（ポートフォワーディング）。
3. **ゲストOS（Ubuntu Server）**
   - **Webサーバ**: Nginx（コンテナ/プロセス用ポート: 80）
   - **自動化**: cronによる定期処理の実行（Pythonスクリプトの自動化）。
   - **リモート管理**: SSHサーバー（ポート: 22）を動作させ、Windowsからの接続を限定受付。
4. **監視プログラム（Python）**
   - `disk_check.py`: `shutil` ライブラリ等を用いてストレージの空き容量・使用状況を収集。
   - 取得結果を自動で `index.html` として出力し、Web公開ディレクトリへ配置。

### 2.2 ポートフォワーディング設計
NAT構成に伴い、ホストOSとゲストOSの間で以下の通信窓口を定義しています。

| サービス名 | ホスト側（Windows） | ゲスト側（Ubuntu） | 用途 |
| :--- | :--- | :--- | :--- |
| **HTTP (Web)** | `localhost:8080` | `ポート 80` | ブラウザからの監視画面閲覧用 |
| **SSH (リモート管理)** | `localhost:2222` | `ポート 22` | コマンドプロンプトからのセキュアな遠隔操作用 |

## 4. テスト・検証項目
- **Nginxサービス稼働確認**: `systemctl status nginx` によるプロセスの正常生存確認。
- **ポートフォワーディング経由の疎通確認**: ホストOS（Windows）のコマンドプロンプトより `ssh vboxuser@localhost -p 2222` でのログイン検証。
- **ブラウザアクセス確認（HTTP疎通）**: Windowsブラウザから `http://localhost:8080` へアクセスし、動的生成されたWebページが表示されることを確認。
- **ディスク情報自動更新確認**: cron実行後にディスク情報が最新状態に更新されているかの確認。
